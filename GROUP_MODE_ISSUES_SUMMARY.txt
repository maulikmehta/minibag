MINIBAG-2 GROUP MODE SESSION JOINING - CRITICAL ISSUES SUMMARY
================================================================

SEVERITY: CRITICAL - Production Bug
FAILURE RATE: 30-80% under concurrent load

QUICK OVERVIEW
==============

7 Critical Bugs Found:

1. RACE CONDITION - Participant Count Check (Line 441-455 in invites/crud.ts)
   Impact: Multiple users can exceed group size limit in concurrent joins
   Fix: Add row-level locking with SELECT ... FOR UPDATE

2. RACE CONDITION - Nickname Claiming Not Atomic (Line 461-487 in invites/crud.ts)
   Impact: Nicknames get orphaned when participant creation fails
   Fix: Move nickname claiming inside join transaction

3. MISSING VALIDATION - Group Mode Not Enforced (Line 1696-1740 in sessions.js)
   Impact: Users can join solo sessions as second participant
   Fix: Add explicit session.mode === 'group' check

4. RACE CONDITION - Duplicate Limit Checks (Sessions API vs sessions-core)
   Impact: Two validation points can make conflicting decisions
   Fix: Consolidate to single validation point with transaction

5. ASYNC SYNC ISSUE - WebSocket Join Not Coordinated (handlers.ts Line 98-137)
   Impact: Broadcast messages may arrive out-of-order, UI shows stale data
   Fix: Ensure join-session handler validates DB state

6. QUERY BUG - Constant Invite Lookup Using Wrong Identifier (Line 403-413)
   Impact: "Invalid invite link" error for valid invites
   Fix: Use direct FK lookup, not nested relation

7. STATE TRACKING - Invite Slot Claims Not Deduplicated (Line 489-501)
   Impact: Same user can claim multiple slots by retrying
   Fix: Check if participant already in slotAssignments
